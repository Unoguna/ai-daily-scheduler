# AI 기반 적응형 일일 스케줄링 코치 시스템

## 1. 프로젝트 개요

- **프로젝트명**: AI 기반 적응형 일일 스케줄링 코치 시스템
- **한 줄 설명**: 사용자의 목표, 상태, 고정 일정을 바탕으로 하루 일정을 생성하고, 대화를 통해 일정 수정까지 지원하는 시스템
- **프로젝트 목적**:
    - 사용자가 매일 직접 시간표를 짜야 하는 부담을 줄인다.
    - 사용자 상태(피곤함, 집중도, 감정 상태 등)를 반영한 일정 추천을 제공한다.
    - 단순 일정 관리가 아니라, 실제 실행 가능한 하루 계획을 돕는 코치형 시스템을 만든다.

## 2. 문제 정의

기존 일정 관리 앱은 사용자가 직접 일정을 입력하고 배치해야 하는 경우가 많다.

하지만 실제로는 사용자의 컨디션, 우선순위, 고정 일정, 목표 진행 상태에 따라 매일 적절한 일정이 달라진다.

이 프로젝트는 다음 문제를 해결하고자 한다.

- 사용자가 매일 계획을 세우는 데 드는 피로
- 계획이 현실적인 컨디션을 반영하지 못하는 문제
- 일정 변경 상황이 생겼을 때 즉시 재조정하기 어려운 문제

## 3. 핵심 아이디어

이 시스템은 크게 두 부분으로 동작한다.

1. **규칙 기반 일정 생성 엔진**
    - 사용자의 고정 일정, 목표, 가용 시간, 우선순위를 바탕으로 기본 일정 초안을 생성한다.
2. **AI 기반 일정 수정 보조**
    - 사용자의 대화 입력과 상태 정보를 바탕으로 일정 변경안을 JSON 형식의 수정 명령으로 생성한다.
    - 실제 일정 반영은 서버 내부의 일정 엔진이 수행한다.

즉, **AI가 직접 일정을 마음대로 만드는 것이 아니라**,

**규칙 기반 엔진이 만든 일정에 대해 AI가 수정 제안을 구조화된 형태로 제공하는 방식**이다.

## 4. 주요 기능

### 4.1 사용자 정보 입력

- 사용자 목표 입력
- 요일별 고정 일정 입력
- 선호 시간대 입력
- 현재 상태 입력 (피곤함, 집중도, 감정 상태 등)

### 4.2 일정 생성

- 하루 단위 일정 자동 생성
- 우선순위와 시간 블록 기반 배치
- 휴식 시간 자동 삽입
- 고정 일정과 충돌 방지

### 4.3 일정 수정

- 대화형 요청으로 일정 수정
- 예: "오늘 너무 피곤해서 공부량 줄여줘"
- AI가 수정안을 JSON Ops 형태로 생성
- 서버가 수정안을 검증 후 반영

### 4.4 피드백 및 개인화

- 사용자의 수행 결과 기록
- 선호 패턴 분석
- 이후 일정 생성 시 반영

## 5. 시스템 차별점

- 단순 캘린더가 아니라 **코치형 일정 관리**를 지향한다.
- 사용자의 상태를 반영해 같은 목표라도 다른 일정 결과를 낼 수 있다.
- 규칙 기반 엔진과 AI 역할을 분리하여 안정성과 설명 가능성을 높인다.
- 대화형 수정 기능으로 사용성이 높다.

## 6. 예상 사용자 시나리오

1. 사용자가 오늘의 상태와 목표를 입력한다.
2. 시스템이 고정 일정과 가용 시간을 바탕으로 하루 일정 초안을 생성한다.
3. 사용자가 생성된 일정표를 확인한다.
4. 사용자가 대화로 수정 요청을 한다.
5. AI가 수정 명령(JSON Ops)을 생성한다.
6. 서버가 이를 검증하고 일정에 반영한다.
7. 사용자는 최종 일정표를 확인하고 실행한다.
